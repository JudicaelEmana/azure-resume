Mise en place d'un réseau local sécurisé avec portail captif (pfSense) et authentification centralisée

Conception et déploiement d'une infrastructure réseau sécurisée pour une entreprise de 150 utilisateurs, combinant un portail captif (pfSense) pour le contrôle d'accès et une authentification centralisée via Windows Server 2012 R2. L'objectif : garantir un accès Wi-Fi/Guest sécurisé tout en simplifiant la gestion des identités. Ce réseau sécurisé a réduit les tentatives d'intrusion de 70% et permis un contrôle granulaire des accès, tout en offrant une expérience utilisateur transparente grâce à l'intégration AD.

Points clés:

  • 🔐 Authentification unique (SSO) via Active Directory (Windows Server 2012 R2)
  • 🌐 Portail captif personnalisé (pfSense) pour les invités et employés
  • ⚡ Filtrage des accès par groupes AD (marketing, RH, direction)
  • 📡 Isolation des réseaux (VLANs pour invités/employés/équipements IoT)

Image liée au projet

Défis rencontrés:

  • 🛠️ Compatibilité pfSense/Windows Server 2012 R2 (NPS Radius)
  • 🔌 Gestion des interruptions pendant le déploiement
  • 👥 Formation des utilisateurs au nouveau portail captif

Axes d'amélioration:

  • 📊 Journalisation avancée avec ELK Stack pour le monitoring
  • 🤖 Automatisation des règles de filtrage via scripts PowerShell
  • 🛡️ Migration vers Windows Server 2022 pour une sécurité renforcée

Compétences demontrées:

  • ✅ pfSense
  • ✅ Windows Server 2012 R2
  • ✅ NPS (Radius)
  • ✅ VLAN
  • ✅ Réseau sécurisé
